Why is JavaScript Popular in Web Development?

Posted on 19 Jun 2014 by Kyle Daugherty

JavaScript (JS) was created in the mid-90s (back in the days of Netscape!). Before that, web pages were static and there was no way for you to interact with the page. JS provides the ability to deliver rich, dynamic web content and has, according to RedMonk, become the most popular programming language.

Let's take a look at some of the pros and cons:

Advantages

  • Easy to Learn: JS has been around for several years so there are numerous resources available online making it easy to learn and implement.
  • Speed: Being client-side, JS is very fast because the script can run immediately instead of having to contact and wait for the server. This also reduces the demand on the website server.
  • Ready to Go: All major browsers have JS interpreters built in and there are no plug-ins to install.
  • Easy to Debug: Using tools like Chrome Dev Tools and FireBug make it easy for developers to debug their code.
  • Versatility JS has primarily been used for front-end development; however, that has changed thanks to server-side solutions like Node.js. Running a Node.js server means startups with small development teams can reallocate front-end developers when needed.

Disadvantages

  • Security Issues: Malicious code can be executed on the users' computer.
  • Inconsistent Rendering: Browsers interpret JS differently resulting in inconsistency and unpredictability. Also, there's a small percentage of users who have JS disabled in their browers.
