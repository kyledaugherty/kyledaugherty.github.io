Git Cheat Sheet

Command Description
git init Creates a new local repository.
git clone [url] Downloads a project and its entire version history from remote repository.
git remote add [bookmark-name] [url] Assigns the original repository to a remote repository that can be used to fetch changes not present in the local repository without modifying your files.
git remote -v Lists all configured remote repositories.
git status Lists all new or modified files and their current state (i.e., tracked, untracked, etc.)
git diff Shows file differences not yet staged.
git diff --staged Shows file differences between staging and the last file version.
git add [filename] Adds the file to the staging area in preparation for versioning. To add all files, use "." in place of the filename.
git reset [filename] Unstages the file without losing its contents.
git rm [filename] Deletes the fiel from the working directory and stages the deletion.
git commit -m "[descriptive message]" Records staged files in version history. Use the -m flag to add a description of the changes.
git push Uploads local commits to the remote repository.
git fetch [bookmark] Downloads all history from the repository bookmark.
git branch Lists all local branches in the current repository.
git branch [branch-name] Creates a new branch.
git checkout [branch-name] Switches to the specified branch and updates the working directory.
git merge [branch] Combines the specified branch's history into the current branch.
git branch -d [branch-name] Deletes the specified branch.
